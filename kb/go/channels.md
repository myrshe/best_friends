# Каналы (Channels) в Go

## Базовая концепция
Каналы — типизированные конвейеры для передачи данных между горутинами. Реализуют философию CSP: *"Не общайтесь через общую память, делитесь памятью через общение"*.

Создание и операции:
```go
ch := make(chan int)           // небуферизированный
bufCh := make(chan string, 10) // буферизированный

ch <- 42        // отправка (блокирует, пока не прочитают)
value := <-ch   // получение (блокирует, пока не отправят)
```

## Типы каналов
| Тип | Поведение | Когда использовать |
|-----|-----------|-------------------|
| Небуферизированный | Синхронный: отправка и чтение блокируют друг друга | Точная синхронизация |
| Буферизированный | Асинхронный: отправка блокируется только при полном буфере | Конвейеры, батчинг |
| `chan<- T` | Только отправка | Типизация воркеров |
| `<-chan T` | Только чтение | Типизация потребителей |

## Паттерны
Worker pool:
```go
func worker(id int, jobs <-chan int, results chan<- int) {
    for job := range jobs {
        results <- job * 2
    }
}
```

Select для множественных операций:
```go
select {
case msg := <-ch1:
    fmt.Println("Получено:", msg)
case <-time.After(1 * time.Second):
    fmt.Println("Таймаут!")
}
```

## Частые ошибки
❌ Отправка в закрытый канал → паника:
```go
close(ch)
ch <- 42 // panic: send on closed channel
```

✅ Правила безопасности:
- Закрывает канал только отправитель
- Используйте `context` для отмены долгих операций
- Проверяйте закрытие: `val, ok := <-ch`
